White Label for Real Estate Agencies

Three approaches to delivering TrendsOnFire reports and market pages under a client's own brand and domain — with no TrendsOnFire logo visible to their clients.

← Back to internal pages

Context: GustoCasa (and future agency clients) on the Full Intelligence plan (€295/month) include a white label option: all reports and market pages delivered under the client's brand and domain. This page documents the three implementation approaches, their complexity, and the recommended path.

Option A
CNAME / DNS Redirect
Simplest setup — domain only, no brand change

The client adds a CNAME DNS record on their domain pointing a subdomain (e.g. markt.gustocasa.nl) to the GitHub Pages URL. The page loads under their domain in the browser address bar.

  • Client domain in URL: yes — visitors see markt.gustocasa.nl
  • TrendsOnFire logo removed: no — same page, same branding
  • Client logo: no

Use case: A quick "looks professional" setup when the client just wants a clean domain. Not a true white label.

Setup time15 min (DNS config)
Maintenance per updateZero — one codebase
True white labelNo
Best forSpeed / test
Option B
Separate Branded Copy of Files
True white label — manual ~5 min per update

Maintain a second copy of the HTML files with the TrendsOnFire logo replaced by the client's logo, client colors applied, and hosted either on GitHub Pages under a custom domain or on the client's own hosting.

  • Client domain in URL: yes
  • TrendsOnFire logo removed: yes
  • Client logo and colors: yes
  • Update process: copy updated file to client folder → push → live within 2–5 min

Practical setup for GustoCasa: Create a gustocasa-wl/ folder on GitHub, apply their logo + colors once. Every monthly update = copy the updated report file into that folder and push. ~5 minutes of work per month.

Setup time2–3 hours (once)
Maintenance per update~5 min/month
True white labelYes
Best for1–2 clients now
Option C
Parameterized Multi-Client System
Automated — one codebase, config file per client

The HTML pages read a client config (URL param or JSON file) and render the correct logo, colors and domain dynamically. Adding a new client = new config entry. Updates deploy once to all clients simultaneously.

  • Client domain in URL: yes (via CNAME per client)
  • TrendsOnFire logo removed: yes
  • Client logo and colors: yes — from config
  • Update process: push once → all white-label clients update automatically
  • Adding a new client: add a config entry + set up their CNAME (~30 min)

Dev effort: ~3–4 hours of engineering to parameterize the existing pages (replace hardcoded logo/color references with config lookups, add a client config JSON). One-time investment.

Setup time3–4 hours dev (once)
Maintenance per updateZero — automatic
True white labelYes
Best for3+ clients

Recommended Path

Now (GustoCasa only): Option B. Set up a branded copy once (~2–3h), then each monthly update costs 5 minutes. Fast to deliver, zero new infrastructure.

When a second agency client signs: Switch to Option C. Build the parameterized system once (~3–4h dev), then every future client is a config entry and zero extra maintenance per update. The automation pays for itself immediately at 2+ clients.
