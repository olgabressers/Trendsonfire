Why real estate agencies specifically? Every Dutch real estate agent needs local market data for three recurring jobs: (1) winning listing meetings against competitors, (2) advising buyers on price and timing, (3) maintaining a professional LinkedIn presence. They currently do this manually — pulling from NVM, Funda, CBS, and their own memory. TrendsOnFire replaces 4–6 hours of monthly research with one automated report delivered on the 12th. The ROI case is immediate and unambiguous.
Segment Analysis — Which Agency to Target First?
Recommendation

Start with boutique independents (1–5 agents). Scale to mid-size. Avoid franchises until year 2.

The boutique segment has the fastest sales cycle (days, not months), the highest pain intensity (owner does everything personally), and the lowest friction to close. One LinkedIn message + one call is enough. At €295/month, the decision is beneath the level of deliberation for any agency earning €200K+ per year. Build 15–20 proof-of-concept clients fast, then use those case studies to move up-market to mid-size agencies at €495–695/month.

Competitive Landscape — Honest Assessment
Important: The underlying data is NOT unique. NVM members already have access to NVM reports, Funda Marktinzicht, Kadaster (public), and CBS statistics. Every agent theoretically has the ingredients. What TrendsOnFire sells is the synthesis, interpretation, and finished content — which no one else provides at the boutique level. The pitch must never be "I have data you can't get." It must be "I turn data you'd never have time to process into something you can use tomorrow morning."

What agents already have access to — and why it's not enough

Tool / SourceWhat it actually providesWhy agents still don't use it effectively
NVM membership (Brainbay) Quarterly PDF reports (national + some regional), transaction data access, AVM valuations. ~€1,500/year included in NVM membership. Quarterly cadence — 3 months out of date. National/regional only — no neighbourhood breakdown. No interpretation, no narrative. No LinkedIn content. Agent still has to do all the analytical work themselves.
Funda Marktinzicht City-level seeker count vs listing count. Demand/supply ratio per city, per property category. Free with NVM membership. Shows one number per city. No price trends, no overbidding, no mortgage rates, no historical view, no neighbourhood breakdown. Starting point, not a product.
Kadaster (public) Full registry of all real estate transactions in NL — prices, addresses, dates. Legally public. Raw database. Requires download, cleaning, and analysis. A boutique owner has no time or skills to build this workflow monthly. Data without context is noise.
CBS Statistics National housing statistics, population data, migration flows. Free, public. National level, quarterly or annual. Far too aggregate to be useful for a single-city boutique agency. No actionable output.
Calcasa AVM (automated valuation model). Used by mortgage lenders and appraisers to value individual properties. Designed for valuation, not market intelligence. Expensive for boutique agencies. No report output, no LinkedIn content, no strategy layer.
DIY research Agent manually pulls from NVM PDF, Funda Marktinzicht, their own transaction memory, local news. Takes 4–6 hours per month. Inconsistent quality. No benchmarking. LinkedIn posts are vague and qualitative. This is what most boutique agents actually do today.

URL Security — Protecting Client Reports
Current problem: URLs like admin/sample-gustocasa.html and gustocasa/market-live.html are guessable by anyone who knows the naming convention. A competitor agency could find a rival's benchmarking data, pricing, and transaction history. This must be fixed before signing more clients.

Solution: random token in URL path

Add an 8-character random token to each client's folder path. No backend required — works on static GitHub Pages hosting. The token is unguessable (36⁸ = 2.8 trillion combinations) and not indexed by Google (noindex meta already set).

TypeCurrent (guessable)Fixed (secure)
Sample reportadmin/sample-gustocasa.htmladmin/clients/a7f3k2b9/report.html
Proposaladmin/gustocasa-proposal.htmladmin/clients/a7f3k2b9/proposal.html
Live dashboardgustocasa/market-live.htmlclients/a7f3k2b9/market-live.html

Each client gets one token for all their URLs. Token is stored in the delisted.html admin registry. This is the same approach used by Dropbox, Google Docs, and Calendly for "anyone with the link" sharing.
